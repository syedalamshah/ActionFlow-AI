# Action Flow Implementation Plan

Action Flow is an Agentic AI System that ingests unstructured business content, extracts insights, analyzes impact, generates prioritized action recommendations, simulates execution, and shows before and after outcomes.

## System Architecture

```mermaid
graph TB
    subgraph Mobile["React Native App (APK)"]
        UI[Dashboard UI]
        Upload[Content Upload]
        Results[Results Viewer]
        Sim[Simulation Viewer]
    end

    subgraph Backend["FastAPI Backend"]
        API[REST API Layer]
        Ingest[Ingestion Engine]
        Agents[Agentic Pipeline]
        Sim_Engine[Simulation Engine]
    end

    subgraph AI["Gemini API"]
        LLM["gemini-2.5-flash"]
    end

    Upload -->|"POST /analyze"| API
    API --> Ingest
    Ingest -->|"Parse and Chunk"| Agents
    Agents -->|"Structured Output"| LLM
    LLM -->|"Insights and Actions"| Agents
    Agents --> Sim_Engine
    Sim_Engine -->|"Mock CRM/Email/Slack"| Sim_Engine
    API -->|"JSON Response"| Results
    API -->|"Simulation Logs"| Sim
```

### Agentic Pipeline (Plan, Execute, Analyze Loop)

```mermaid
sequenceDiagram
    participant User
    participant API as FastAPI
    participant P as Planner Agent
    participant E as Extractor Agent
    participant A as Analyzer Agent
    participant R as Recommender Agent
    participant S as Simulator Agent

    User->>API: Upload content
    API->>P: Route to Planner
    P->>P: Classify content type and plan extraction strategy
    P->>E: Dispatch extraction task
    E->>E: Call Gemini to extract entities and metrics
    E->>A: Pass structured insights
    A->>A: Call Gemini to score impact and identify risks
    A->>R: Pass analysis results
    R->>R: Call Gemini to generate prioritized actions
    R->>S: Pass action recommendations
    S->>S: Simulate each action
    S->>API: Return full pipeline result
    API->>User: Insights, Actions, and Simulated Outcomes
```

## User Review Required

> [!IMPORTANT]
> **Gemini API Key**: A valid `GEMINI_API_KEY` is required. The plan uses `gemini-2.5-flash` for speed and cost efficiency.

> [!IMPORTANT]
> **React Native APK Build**: Building an APK requires either EAS Build or a manual Gradle build. Confirm the preferred build approach.

> [!WARNING]
> **Scope**: The simulation engine uses mock API calls. The before and after outcomes are AI projected.

## Proposed Changes

### Component 1: Backend Foundation

#### [NEW] requirements.txt
Dependencies:
* `fastapi>=0.115.0`
* `uvicorn[standard]>=0.30.0`
* `google-genai>=1.0.0`
* `pydantic>=2.0`
* `pydantic-settings>=2.0`
* `python-multipart>=0.0.9`
* `pandas>=2.0`
* `python-dotenv>=1.0`
* `httpx>=0.27`

#### [NEW] config.py
* `Settings` class using `pydantic-settings` for `GEMINI_API_KEY`, `GEMINI_MODEL`, `API_HOST`, `API_PORT`
* Load from `.env` file

#### [NEW] main.py
* FastAPI app initialization with CORS middleware
* Include v1 router
* Startup and shutdown lifecycle events
* Health check at root

### Component 2: API Layer

#### [NEW] request.py
Pydantic models:
* `AnalyzeRequest`: `content: str`, `content_type: Literal["sales_report", "news", "csv", "mixed"]`, `context: Optional[str]`
* `FileUploadRequest`: For multipart file uploads

#### [NEW] response.py
Pydantic models:
* `Insight`: `title`, `description`, `category`, `confidence_score`, `source_snippet`
* `ImpactAnalysis`: `risk_level`, `opportunity_score`, `revenue_impact`, `urgency`
* `ActionRecommendation`: `priority: int`, `title`, `description`, `action_type`, `expected_outcome`, `effort_level`
* `SimulationResult`: `action_title`, `simulated_service`, `status`, `mock_response`, `timestamp`
* `BeforeAfterOutcome`: `metric`, `before_value`, `after_value`, `change_pct`, `explanation`
* `FullAnalysisResponse`: Combines all of the above into the pipeline output

#### [NEW] analyze.py
* `POST /api/v1/analyze` accepts text content or file upload, runs the full agentic pipeline, returns `FullAnalysisResponse`
* `POST /api/v1/analyze/file` accepts file upload
* `GET /api/v1/analyze/{job_id}/status` check pipeline status

### Component 3: Agentic Pipeline

#### [NEW] gemini_client.py
* Wraps `google.genai.Client`
* `async def generate_structured(prompt, schema)` calls Gemini with `response_mime_type="application/json"` and Pydantic `response_schema`
* Handles retries and error mapping

#### [NEW] planner.py
Planner Agent classifies input content:
* Detects content type
* Identifies key entities and metrics present
* Creates an extraction plan
* Output: `ExtractionPlan` Pydantic model

#### [NEW] extractor.py
Extractor Agent pulls structured insights from raw content:
* Prompt: Extract key business insights, metrics, trends, and signals from this content
* Uses Gemini structured output to return `list[Insight]`
* Handles CSV data by converting to natural language summary first

#### [NEW] analyzer.py
Analyzer Agent evaluates business impact:
* Takes extracted insights as input
* Prompt: Analyze the business impact of these insights. Score risk, opportunity, revenue impact
* Output: `ImpactAnalysis` per insight
* Identifies cross-cutting themes and correlations

#### [NEW] recommender.py
Recommender Agent generates prioritized actions:
* Takes insights and impact analysis as input
* Prompt: Generate 3 to 5 prioritized action recommendations
* Each action has priority rank, description, expected outcome, effort level, action type
* Output: `list[ActionRecommendation]`

#### [NEW] simulator.py
Simulator Agent mock executes each action:
* For each recommendation, simulates the appropriate integration
* CRM Update mock Salesforce API call with response
* Email Outreach mock SendGrid API call
* Slack Notification mock Slack webhook
* Strategy Document mock Google Docs creation
* Generates before and after projections using Gemini
* Output: `list[SimulationResult]` and `list[BeforeAfterOutcome]`

#### [NEW] pipeline.py
Orchestrator runs the full Plan, Extract, Analyze, Recommend, Simulate loop:
* Accepts raw input
* Calls each agent sequentially, passing outputs downstream
* Includes timing and logging for each stage
* Returns `FullAnalysisResponse`

#### [NEW] mock_integrations.py
Mock service responses:
* `mock_crm_update()` returns fake Salesforce response JSON
* `mock_send_email()` returns fake SendGrid response
* `mock_slack_notify()` returns fake Slack webhook response
* `mock_create_document()` returns fake Google Docs link
* Each includes realistic latency simulation

#### [NEW] file_parser.py
* `parse_csv(file)` reads CSV with pandas, converts to text summary
* `parse_text(file)` reads plain text files
* `detect_content_type(content)` heuristic detection

### Component 4: Sample Data

#### [NEW] sales_report.txt
Synthetic quarterly sales report with realistic metrics, team performance, regional breakdown, and pipeline data.

#### [NEW] quarterly_data.csv
CSV with columns Month, Region, Revenue, Deals Closed, Pipeline Value, Win Rate, Churn Rate.

#### [NEW] industry_news.txt
Collection of synthetic industry news snippets about market trends, competitor moves, regulatory changes.
